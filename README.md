# 🤖 Procurement AI Assistant
Интеллектуальный помощник для отдела закупок на базе GigaChat от Сбера. Автоматизирует обработку коммерческих предложений, сравнивает поставщиков и помогает готовить ответы.

## Что нового в этой версии
Мы значительно улучшили производительность и добавили инструменты для мониторинга:

## Ускорение работы
GPU-ускоренный векторный поиск – теперь семантический поиск по базе предложений может использовать видеокарту (CUDA) для кратного ускорения (класс AcceleratedVectorSearch).

Кэширование Redis – ответы GigaChat и результаты поиска сохраняются в Redis, что сокращает время повторных запросов и снижает нагрузку на API.

## Логирование производительности
BenchmarkLogger – автоматический замер времени выполнения всех ключевых операций (загрузка модели, генерация эмбеддингов, запросы к GigaChat, обработка файлов и т.д.).
Отдельная вкладка "Производительность" – наглядное отображение статистики с возможностью очистки логов.

## Управление кэшем
В боковой панели появился раздел "Кэш Redis" с индикатором подключения и кнопкой полной очистки кэша.

## О проекте
Procurement AI Assistant — это веб-приложение, созданное для автоматизации работы отдела закупок. Оно использует нейросеть GigaChat от Сбера для:
Извлечения структурированных данных из коммерческих предложений (PDF, DOCX, TXT)
Сравнения нескольких предложений и выбора лучшего
Генерации профессиональных ответов поставщикам
Семантического поиска по базе предложений
Приложение разработано специально для российских компаний с учетом требований законодательства и использованием отечественных технологий ИИ.

## Функциональность
### Обработка входящих предложений
* Загрузка файлов в форматах PDF, DOCX, TXT, CSV

### Автоматическое извлечение:
* Название поставщика и ИНН
* Список товаров с количеством и ценами
* Общая сумма предложения
* Условия доставки и оплаты
* Контактные данные
* Ключевые преимущества
* Сохранение в локальную базу данных (JSON)

### Сравнение предложений
* Выбор 2-4 предложений для сравнения
* AI-анализ с определением лучшего варианта
* Выявление рисков по каждому поставщику
* Рекомендации по выбору
* Оценка возможной экономии
* Формирование списка вопросов для переговоров

### Генерация ответов
* Создание профессиональных деловых писем
* Учет специфики предложения
* Настройка тона письма (официальный/деловой/дружелюбный)
* Копирование в буфер обмена
* Отправка через почту (mailto ссылка)

### Семантический поиск
* Поиск по смыслу, а не по ключевым словам
* Векторная индексация всех предложений
* Быстрый доступ к истории
* Визуализация степени схожести результатов

### Интеграция с почтой (опционально)
* Автоматическая проверка входящих писем (IMAP)
* Обработка вложений из писем
* Пометка прочитанных писем

## Быстрый старт
```
# 1. Клонируйте репозиторий
git clone https://github.com/yourusername/procurement-ai-assistant.git
cd procurement-ai-assistant

# 2. Создайте виртуальное окружение
python -m venv venv
source venv/bin/activate  # для Linux/macOS
# venv\Scripts\activate   # для Windows

# 3. Установите зависимости
pip install -r requirements.txt

# 4. Для запуска Redis на локальном стенде в консоли прописать следующие команды:
docker build -t my-redis .
docker run -d --name redis-server -p 6379:6379 my-redis
## 4.1 Для проверки работоспособности можно запустить тестовую команду
docker run -d --name redis-server -p 6379:6379 my-redis # Должен вернуться PONG  

# 5. Запустите приложение
streamlit run main.py

# Примеры  коммерческих предложений есть в папке "примеры Ком предложений"
```
